SECTION 1 : SYSTEM WORKING OVERVIEW (सिस्टीम कशी काम करेल)
================================================================================

Junior Aladdin ही फक्त NIFTY 50 साठीची एक auto market-observation + trading
system आहे. ती एका professional trader च्या मेंदूसारखी काम करते - पण ती एक
technical brain आहे, म्हणून ती एका माणसापेक्षा जास्त data एकाच वेळी बघू शकते.

मूळ कल्पना (एका वाक्यात):
  Market मधून सगळा data घ्यायचा -> तो स्वच्छ करायचा -> त्यावर professional
  trader जसे analysis करतात तसे सगळे calculations करायचे -> ते वेगवेगळ्या
  तज्ज्ञांनी (Department Heads) वाचून प्रत्येकाने आपले मत बनवायचे -> ती सगळी
  मते एका राजाने (Captain) ऐकायची, एकत्र करायची, आणि एकच निर्णय घ्यायचा:
  trade करायचा की नाही. निर्णय झाला तर तो Execution कडे जातो (alert/paper/real).
  आणि सगळे काही Dashboard वर दिसते.

System चे स्वभाव (philosophy):
  - Survival First : प्रथम पैसे न गमावणे, मग नफा. flat दिवस = यशस्वी दिवस.
  - Confluence or Silence : अनेक गोष्टी एकाच दिशेला जुळल्या तरच trade.
  - Context First : trade आधी market ची आजची स्थिती (regime, session, story).
  - Trap Awareness : प्रत्येक signal वर "हा सापळा तर नाही?" हा प्रश्न.
  - Discipline : loss नंतर लहान/थांब, कधीही revenge/greed नाही.
  - तिन्ही mode (ALERT/PAPER/REAL) मध्ये सारख्याच seriousness ने काम.

Captain चा विचार करण्याचा rhythm (दोन-टप्पा / Two-Speed):
  - HEAVY (candle close वर): पूर्ण इतिहास वाचून जड analysis. Heads आपापले
    reports बनवतात, Captain एक "तयार योजना" (armed plan) बनवतो - म्हणजे
    "किंमत X ला आली + अट पूर्ण झाली तर हा trade."
  - LIGHT (प्रत्येक tick वर): फक्त बघायचे की armed plan ची अट पूर्ण झाली का.
    झाली -> instant trade trigger. नाही -> वाट बघ.
  कारण: प्रत्येक tick ला जड analysis केले तर laptop hang होईल आणि अर्ध्या
  candle चा खोटा signal येईल. म्हणून निर्णय candle वर, पहारा tick वर.
  फायदा: वेळ निघून जात नाही (tick trigger instant) + system hang होत नाही
  + मानसिकता प्रत्येक tick ला बदलत नाही (discipline टिकते).

SECTION 3 : DATA FLOW - MARKET TO TRADE (पूर्ण cycle)
================================================================================

एका सामान्य trading दिवशी data कसा वाहतो:

  STEP 1  MARKET
          NIFTY 50 spot + त्याचे ATM/ITM/OTM premiums + option chain.

  STEP 2  FLOOR 1 - MARKET CONNECTION (fetch)
          Angel One WebSocket -> live ticks (LTP, OHLC, volume, depth)
          Angel One REST -> option chain, OI, historical, VIX
          बाहेरून -> FII/DII, Global cues, Calendar (holiday/expiry/event)
          OUTPUT -> raw data Floor 2 कडे.

  STEP 3  FLOOR 2 - DATA CENTER (clean + store)
          raw store (जसाच्या तसा) -> validate (duplicate, timestamp,
          missing, corrupt candle) -> clean -> structured store
          (major / minor / computed) + quality score.
          OUTPUT -> clean, verified data Floor 3 कडे.

  STEP 4  FLOOR 3 - CALCULATION ENGINES
          clean data वर सगळी आकडेमोड:
          structure (swing, BOS, CHoCH, trend), SMC (OB, FVG, liquidity,
          sweep), ICT (premium/discount, displacement), price action
          (candle patterns), technical (EMA, RSI, VWAP, ATR, ADX,
          supertrend), options (PCR, OI, max pain, GEX, IV, greeks),
          macro (VIX, FII/DII, global).
          OUTPUT -> calculated values Floor 4 कडे.

  STEP 5  FLOOR 4 - DEPARTMENT HEADS (6 तज्ज्ञ)
          प्रत्येक Head आपापली calculations वाचतो + संपूर्ण इतिहास बघतो
          (फक्त चालू candle नाही) + स्वतःची memory ठेवतो (कोणता
          FVG/OB/sweep अजून जिवंत आहे) आणि एक report बनवतो:
             bias, confidence, reasons, timeframe_view,
             active_zones, armed_triggers, invalidation, timestamp.
          (Psychology Head वेगळा: bias देत नाही, फक्त allow/block + brake.)
          OUTPUT -> 6 reports Floor 5 (Captain) कडे.

  STEP 6  FLOOR 5 - CAPTAIN (राजा)
          HEAVY (candle close): 6 reports + context घेऊन 6-layer विचार
          करतो (खाली SECTION 6 मध्ये detail येईल) आणि armed plan बनवतो.
          LIGHT (प्रत्येक tick): trigger अट पूर्ण झाली का बघतो.
          निर्णय: TRADE / WAIT / BLOCKED. trade असेल तर entry, SL,
          target, size ठरवतो (तुझ्या manual capital च्या आत).
          OUTPUT -> निर्णय Side A (Execution) कडे.

  STEP 7  SIDE A - EXECUTION
          ALERT  -> Telegram notification (तिन्ही mode मध्ये नेहमी).
          PAPER  -> simulated trade (तू सेट केलेल्या खोट्या capital ने,
                    खरी slippage + cost वजा करून).
          REAL   -> Angel One ला limit/SL-limit order (तू सेट केलेल्या
                    खऱ्या capital ने).
          सगळे log book मध्ये नोंद (वेळ, entry, SL, target, P&L).

  STEP 8  SIDE B - DASHBOARD
          वरचे सगळे live दाखवतो. तू इथे mode (PAPER/REAL), capital,
          kill-switch नियंत्रित करतोस. ती सेटिंग पूर्ण system मध्ये sync होते.

  STEP 9  POSITION MANAGEMENT (trade उघडल्यावर)
          प्रत्येक tick ला उघड्या position चा SL/target/trailing/time-exit
          बघितला जातो. thesis invalidate झाला तर बाहेर. EOD ला बंद.

  परत STEP 2 पासून cycle चालू राहते (दर tick / दर candle).
